# Git/GitHub Practice (3 Questions)

이 문서는 Git/GitHub 협업 흐름을 직접 수행하며 익히기 위한 연습 문항이다.  
모든 답안은 `practice/` 폴더에 제출한다.

---

## 제출 규칙

1. `practice/` 폴더 안에 본인 파일을 생성한다.
- 파일명 형식: `이름-학과.md`
- 예시: `홍길동-컴퓨터공학.md`

2. 캡처 이미지는 `practice/images/`에 저장하고, 본인 문서에 링크로 첨부한다.
- 파일명 형식: `이름-q1.png`, `이름-q3.png`
- 예시: `홍길동-q1.png`, `홍길동-q3.png`

3. 모든 문항의 결과는 본인 문서(`이름-학과.md`)에 정리한다.

---

## 1. 브랜치 분기/병합 및 로그 그래프 제출

소스 코드를 수정하여 다음 과정을 수행한다.

- 최초로 Git을 초기화한다.
- 최소 2개의 브랜치를 생성하여 각각 다른 변경사항을 커밋한다.
- 최종적으로 main(또는 master) 브랜치로 병합한다.
- 아래 명령의 결과를 실행하고, 그래프가 보이도록 캡처하여 첨부한다.

실행 명령
- git log --oneline --graph --all

제출물
- `practice/images/이름-q1.png` 캡처 파일
- 본인 문서에 캡처 이미지 링크 포함

---

## 2. Fork 기반 제출 및 자기소개 PR 병합

다음 과정을 수행한다.

- 이 GitHub 레포지토리를 Fork 한다.
- Fork한 레포지토리를 기반으로 본인 파일을 생성한다.
  - 위치: `practice/이름-학과.md`
- 파일에 간단한 자기소개를 마크다운 형식으로 작성한다.
  - 필수 포함: 이름, 학과, 관심 분야, 하고 싶은 프로젝트/목표(1줄)
- 본인 Fork에서 원본 레포지토리로 Pull Request(PR)를 생성하고 병합한다.

제출물
- `practice/이름-학과.md` (자기소개 포함)
- PR 링크를 본인 문서에 함께 기입

---

## 3. Issue 생성 + PR 승인(Approve) 후 병합 (컨벤션 필수)

다음 과정을 수행한다.

- 이 레포지토리에 Issue를 생성한다.
- 2번에서 만든 본인 문서(`practice/이름-학과.md`)에 추가 수정사항을 반영한다.
- Pull Request를 생성하고, 리뷰어에게 승인(Approve)을 받은 뒤 병합한다.

3번은 [컨벤션](99-collaboration-convention-definition.md)을 반드시 따른다.


제출물
- Issue 링크를 본인 문서에 기입
- PR 링크를 본인 문서에 기입
- 승인(Approve)이 확인되는 화면을 캡처하여 첨부(권장)
  - `practice/images/이름-q3.png`
